Conversão de tipo: O ID da resposta do Forms é texto; o ID na base de medição é número. O PROCV usa VALOR() para converter texto → número inteiro antes da procura.

Ajuste de horário: A automação registra hora com offset de +3h em relação ao real. Subtração de 0,125 (fração de dia = 3h) corrige o problema. Método: comparação manual entre hora de chegada física e hora registrada, até convergir.

Fórmulas de faturamento: SOMASE aninhado procura por cliente + período + serviço e retorna volume em m³. PROCV de apoio retorna valor unitário. Descontos aplicados diretamente no unitário antes da multiplicação.
Domingo / Sem acesso ao Forms: Porteiro não gera ID digital. Equipe de operação preenche via impressa manualmente. Impacto: perda de rastreabilidade digital, inserção manual posterior.

Entrada sem faturamento: Quando caminhão não descarta (flat de gordura, porta lançada 2x). Requer observação justificando. Check administrativo identifica esses casos.

Faturamento sem entrada: Anomalia que exige investigação com equipe de recebimento.
Lista de Ordem: Após clientes confirmarem (ou após 2 dias sem retorno), copia-se manualmente as colunas I a AD da planilha de faturamento. Solicitação via Servelo: Catálogo → Faturamento → Lista de Ordem. Campos fixos: contrato, elemento de custo. Anexa planilha e informa referência (ex: "Lista de Ordem nº 142").

Controle Financeiro: Planilha gerida por Alessandro. Campos: tipo de movimentação (entrada, parcelamento, adiantamento), número NF, parcela atual / total, CNPJ, data de lançamento, valor emitido, vencimento, dias bloqueados, data de bloqueio, tratativa, data de pagamento, valor recebido, juros, receita/despesas. Fórmula de status: se vencido, calcula diferença entre data de bloqueio e vencimento. Campos de receita/despesas raramente preenchidos na prática.
Conceito: Digitalizar todos os MTRs do dia no final do expediente. Automação (Power Automate + OCR/IA) lê os documentos e extrai: número MTR, razão social do gerador, CNPJ, transportador, placa, motorista, código IBAMA, volume (litros/m³/ton).

Desafios identificados: Os MTRs são PDFs de imagem (não copiáveis). A qualidade da cópia varia e causa dificuldade para OCR reconhecer números. O campo de assinatura do motorista é manuscrito, não digitado — difícil de extrair automaticamente.

Via alternativa (QR Code): Cada MTR possui QR Code que redireciona ao site do IMA com o número do MTR já preenchido na tela de consulta. Possível via de extração de dados via scraping ou API do IMA — mas até o momento não há API disponível.

Seção 04
Matriz de Riscos
Troca de placas entre fornecedores
ALTO
Formulário permite selecionar qualquer placa para qualquer fornecedor. Com dezenas de placas semelhantes, erro é iminente. Impacto: faturamento incorreto, rastreabilidade comprometida.
Etapa: P-02 · Ator: Porteiro
Indisponibilidade do formulário
ALTO
Sem acesso ao Forms (domingo, problema técnico), não se gera ID digital. Operação preenche via impressa. Perda de rastreabilidade e inserção manual posterior.
Etapa: P-02 · Ator: Porteiro / Operação
Offset de horário empírico
MÉDIO
Fração 0,125 encontrada por tentativa e erro. Sem documentação da causa raiz. Mudança na plataforma pode invalidar a correção sem aviso.
Etapa: A-02 · Ator: Automação
Erro no lançamento IMA
MÉDIO
Busca manual por destinador no site é demorada e propensa a erros. Volume diário amplifica o risco.
Etapa: M-02 · Ator: Administrativo

Seção 05
Melhorias Priorizadas (TO-BE)
IMPACTO: ALTO ESFORÇO: S
1. Geração automática de ID no Forms
Elimina sequencial manual. ID aparece imediatamente para o porteiro. Remove dependência do recebimento para associar ID real. Resolve risco de domingos.
Resolve: P-01, P-02
IMPACTO: ALTO ESFORÇO: S
2. Filtro condicional de placas por fornecedor
Ao selecionar fornecedor, exibir apenas suas placas vinculadas. Elimina risco de associação incorreta. Base de placas já existe.
Resolve: P-02
IMPACTO: ALTO ESFORÇO: M
3. Automação de envio de BM por e-mail
Gerar e disparar BM automaticamente para cada fornecedor no fechamento do período. Principal gargalo manual citado na transcrição.
Resolve: F-01, F-02
IMPACTO: MÉDIO ESFORÇO: XS
4. Correção de timezone na automação
Investigar causa raiz do offset de 3h. Configurar timezone correto na plataforma em vez de subtração empírica.
Resolve: A-02
IMPACTO: ALTO ESFORÇO: S
5. Automação da cópia de dados para Lista de Ordem
Automatizar extração das colunas I–AD do faturamento para lançamento de Lista de Ordem no Servelo. Processo atualmente manual e repetitivo.
Resolve: NF-01
IMPACTO: ALTO ESFORÇO: L
6. OCR + extração de dados do MTR
Digitalizar MTRs e usar OCR/IA para extrair dados (número, CNPJ, placa, volume). QR Code no MTR leva ao IMA com número preenchido — possível via alternativa. Desafios: PDFs de imagem, qualidade variável.
Resolve: M-02
