VERSÃO 1.0 — ESCOPO BASE 120h
1 Análise, diagnóstico e modelagem do processo ANÁLISE Documentação, Mermaid Média 24h
Ingestão de transcrição operacional (~2500 palavras), extração semântica, diagnóstico do fluxo atual (AS-IS), mapeamento de atores, sistemas, riscos, automações existentes e oportunidades de melhoria. Modelagem inicial do workflow em Mermaid. Validação de fidelidade à fonte.
2 Desenvolvimento do Workflow Board operacional DESENVOLVIMENTO HTML, CSS, Mermaid Alta 44h
Construção do painel operacional interativo com swimlanes por área, nós tipados (7 tipos), badges semânticos, tabela de automações, matriz de riscos, cards de melhoria priorizados, painéis expansíveis com detalhes técnicos. Design system com paleta institucional Sanama (6 tokens). Diagrama Mermaid com convenções BPM. CSS modular (~400 linhas).
3 Estruturação do repositório e publicação DESENVOLVIMENTO Repositório, Deploy, Docs Média 22h
Repositório com 17 arquivos: página principal, página 404, favicon SVG, .gitignore, robots.txt, sitemap.xml, workflow de deploy via GitHub Actions, README profissional, guia de publicação, documentação de arquitetura. Pacote validado e publicado.
4 Arquitetura modular do projeto ARQUITETURA Estrutura, CSS, HTML, Docs Média-Alta 30h
Organização em 6 camadas com separação de responsabilidades: app/ (CSS modular em 3 arquivos: tokens, layout, componentes), content/ (dados estruturados em JSON), diagrams/ (Mermaid por fluxo), brand/ (assets institucionais), docs/ (guias técnicos), .github/ (deploy). Estrutura projetada para manutenção contínua e escalabilidade.
Subtotal v1.0 120h
VERSÃO 1.1 — EVOLUÇÃO INCREMENTAL 60h
5 Incorporação dos fluxos de NF e Controle Financeiro EVOLUÇÃO HTML, Mermaid, JSON Média 38h
Classificação e integração de novos sub-processos a partir de segunda transcrição: Nota Fiscal, Controle Financeiro, proposta de OCR/MTR. 4 nós novos no fluxo, 2 swimlanes adicionais, 2 painéis expansíveis, 2 automações no inventário, 2 melhorias priorizadas (IMP-5, IMP-6). Diagrama Mermaid estendido. Dados estruturados atualizados.
6 Validação de qualidade e consistência QUALIDADE HTML, JSON Alta (atenção) 22h
Validação cruzada de consistência entre todas as camadas do projeto (interface, dados estruturados, diagramas). Ajuste fino de métricas, classificações e indicadores. Verificação de integridade do pacote para publicação final.
Subtotal v1.1 60h

Premissas
Premissas técnicas do escopo
Perfil técnico: Profissional sênior com domínio em front-end, modelagem de processos (BPM), GitHub Pages e design de workflows operacionais.

Escopo inclui: Análise de transcrições operacionais, extração de regras de negócio, modelagem de processo, desenvolvimento de interface interativa, codificação HTML/CSS/JS, criação de diagramas Mermaid, estruturação de repositório, documentação técnica, deploy e validação de qualidade.

Escopo não inclui: Reuniões de alinhamento, revisões externas, aprovações, tempo de espera por stakeholders ou implementação em ferramentas de terceiros (Power Automate, Servelo, IMA, sistemas bancários).

Versionamento: v1.0 cobre o escopo base (análise, desenvolvimento, repositório e arquitetura). v1.1 cobre a evolução incremental (novos fluxos de NF e Controle Financeiro) e a validação final de qualidade.
